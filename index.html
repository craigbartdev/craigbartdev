Dating App (View Here)

.NET 5, Angular 10, Entity, PostgreSQL

This is an extended version of the app from the "Building an app with ASP.NET Core and Angular" Udemy course by Neil Cummings. The course covers SignalR chat and presence tracking, Identity authentication, policy based authorization, cloud image storage with Cloudinary, and deployment with Heroku. It also showed me how to run a PostgreSQL database in Docker and manage it with PgAdmin. I wanted to test my knowledge of Angular and .NET 5, so I added a refresh token, forgot password, and email confirmation functionality after completing the course. I also fixed some bugs that were in the original app. This app helped me get comfortable connecting SPAs and APIs, working with branches in Git, and preparing a fullstack app for deployment.

Vidly

.NET Framework, SQL Server, JQuery, Azure

This is also an app that I extended from the "The Complete ASP.NET MVC 5 Course" Udemy course by Mosh Hamedani. It was my introduction to ASP.NET, Visual Studio, SQL Server, and Azure deployment. It is a movie rental app, where customers can rent movies, and users can edit those rentals. This app uses .NET Framework 4.6.1 MVC with Razor Views, and includes OAuth and role based authorization. It also uses WebAPI and JQuery to display data in a third-party table component, and send delete requests. I added the ability for users to mark customers as delinquent on payment, apply discount rates, and limit the number of movies that a customer can rent at one time. I also added an "Admin" role for managing the roles of other users.

Video Title Analyser

MEAN Stack, IBM Watson

This was done for a MEAN stack class at Boston University. The class required the use of two third-pary APIs, and OAuth implementation. I used Google OAuth and the YouTube API to allow a user to log into their Google account and do a YouTube search. The top search results would appear, with each title getting fed through the IBM Watson Natural Language Understanding API, which would analyze the words in the title and display scores between 0 and 1 for different emotions. For example, a search for "Ugly Spider" would return a result with that is high in "Disgust" and low in "Joy". This was my first fullstack app, and we were only required to get the app to work in a local development environment. I remember the asynchronicity of Node.js being a challenge to work with, but everything worked in the end.

React Personal Site

React, .NET Framework, SQL Server

This was my attempt at creating a functional resume/blog application using React and ASP.NET 4.6.1 WebAPI. I built a basic front end in React with a full blog section, authentication, and commenting. I used React Hooks to maintain state in the app and just kept the data stored in localStorage during frontend development. I eventually tried to connect the app to ASP.NET WebAPI and realized that I did not have the knowledge to properly work with both a SPA and an API, particularly when it came to authentication and authorization. It motivated me to take the Neil Cummings ASP.NET Core/Angular 10 course in order to learn. Failing with this app helped me to understand my own limitations, and inspired me to improve and develop a modern skillset. I now feel comfortable working with APIs and SPAs.
